# 単語本の構造（LLM構造化プロンプトの前提知識）

対象の本: **新HSK1~4級単語トレーニングブック**（中国語単語、説明は日本語）。
OCR結果を LLM で vocab-trainer 用 word-drafts JSON に構造化する際の判断基準。

**アップロード単位**: 原則として**1つの Unit・セクション単位**（複数ページ）でまとめてアップロードする。構造化は1回のLLM呼び出しで**全ページをまとめて**行い（各ページは `=== PAGE: <ファイル名> ===` マーカーで区切って渡す）。単語表はページをまたいで続くため、ページ末尾の単語行の例文（や折り返した意味）が**次ページ冒頭に見出し語なしで続く**場合は、同じ単語候補にたたみ込み、またいだページを `sourceImages`（複数可）に記録する。座標 `y` はページごとにリセットされるため、「y が重なる＝同じ行」の判定は同一ページ内に限る。

## 全体構造

- 本は複数の **Unit** で構成される。ページの左端・右端に「UNIT N」と印字されることがあるが、**無視する**。
- 各 Unit には複数の**テーマ**がある。整数（1, 2, 3, ...）で数えられ、この整数は**ページで最も大きい（またはそれぐらいの）サイズ**で印字される。テーマにはタイトルが付くが**無視してよい**。
- 各テーマには複数の**セッション**がある（ない場合もある）。`整数-整数`（例: 2-3, 14-1）で数えられる。セッションのタイトルも**無視してよい**。

## 単語表（3列）

各セッションには表形式で単語が入っている。基本3列:

### 1列目（最も左）: 番号

- 番号の隣に □、□ の下に ★ が入っている。
- □ や ★ は OCR で文字として検出されたりされなかったりするが、**番号とともに基本無視**（行の区切りの手掛かりにのみ使う）。

### 2列目（真ん中）: 中国語単語とその意味

上から順に（基本 1 の下に 2、2 の下に 3）:

1. **中国語単語** — セル内でサイズが最も大きい。
2. **ピンイン** — 単語の隣。
3. **日本語の意味** — 複数の意味がある場合もある。「動」「名」など品詞を表すマーカーが出てくる場合も多々ある。
   - 意味が長くセル幅に収まらない場合は行が変わり、**別のOCRボックスに入ることがある**ので注意。

### 3列目（右）: 例文

- **一つの単語行に複数の例文セルが入ることが多々ある**。
- 各例文セルの構造（2 は 1 の右側または下、3 は 1/2 または 2 の下）:
  1. **中国語例文** — 同セル内でサイズが最も大きい。
  2. **ピンイン** — 1 の右側に位置したり、下に位置したりする。
  3. **日本語訳**

## 行の対応と注釈

- 基本、一つの単語行には一つの番号セルが来る。一つの番号セルにはほとんど一つの中国語単語セルだけがある。
- たまに単語セルの下に「”学び合う”という意味がある」など、その単語に対する**注釈**がある → 定義に適切に反映してよい（**単語の定義には改行ができる**）。
- 一つの単語セルに複数の例文セルが対応することを忘れない。

## 例文の分かち書き（segments）

中国語例文を単語の例文として取り込むときは、**必ずピンインの分かち書きを参考にして文章も分かち書きする**。

- ピンインは異なるボックスに分かれたり、一つのボックス内でも分かち書きされたりするので注意。
- 単語・文法要素ごとにスペースで区切られているので、それをもとに segment を分離し、draft フォーマットの `examples[].segments` の各要素として登録する。

## 単語比較行

各単語行の間には、たまに「これまで出てきた単語」と「最近出てきた単語」を比較して違いを説明する行がある。大体以下の形式の文章が上から下に登場する（各文章が改行して複数ボックスになる場合もある）:

```
（中国語単語）：日本語説明
```

→ 比較対象の単語がこのページの見出し語にあれば、その単語の定義に説明を追記する。ページ上にない単語の行は無視する。
